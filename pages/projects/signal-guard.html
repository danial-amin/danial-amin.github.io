SignalGuard Observability Platform

The problem

Teams need to show how monitoring, metrics, and early AIOps fit together‚Äîwithout running production systems. A coherent demo (microservices ‚Üí Prometheus ‚Üí anomaly detection ‚Üí dashboards) is hard to spin up and explain in interviews or pitches.

The solution

SignalGuard is a full observability demo: synthetic FastAPI microservices emit Prometheus metrics, a separate anomaly-detection service scores them, Prometheus and Grafana are provisioned via Docker, and a standalone GitHub Pages dashboard simulates live metrics and anomalies‚Äîno backend required for the static demo.

Without SignalGuard

Ad-hoc scripts or production-only setups; no single story for ‚Äúmetrics + anomaly + dashboards.‚Äù

With SignalGuard

One repo: microservices, Prometheus, Grafana, anomaly detector, and a static demo dashboard‚Äîinterview- and demo-ready.

What it does

  • Synthetic microservices ‚Äì FastAPI apps with latency, error rate, throughput, and business metrics exported to Prometheus.
  • Anomaly detection service ‚Äì Queries Prometheus, computes anomaly scores, emits flags and scores for dashboards.
  • Prometheus & Grafana ‚Äì Scrape configs and prebuilt dashboards (latency p95, error rate, throughput, anomaly score).
  • GitHub Pages dashboard ‚Äì Static, simulated metrics and anomalies with Chart.js; no backend; ideal for demos.

Tech stack

FastAPI, Prometheus, Grafana, Python (anomaly detector), Chart.js (static dashboard), Docker Compose.
